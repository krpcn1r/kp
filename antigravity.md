# Документация архитектуры проекта: АРМ (Консольное приложение на C++)

## 📌 Обзор проекта
Данный проект представляет собой Автоматизированное Рабочее Место (АРМ), реализованное как консольное приложение на C++. Система предназначена для управления пользователями и клиентской базой. Данные хранятся в формате JSON.

## 📂 Структура каталогов

### 📂 `admin/` (Панель администратора)
* `AdminPanel.cpp / .h` — Управление главным меню администратора, распределение прав.
* `ConfigWizard.cpp / .h` — Мастер первого запуска (создание первого админа, настройка путей).
* `LogViewer.cpp / .h` — Просмотр действий пользователей и системных ошибок.

### 📂 `auth/` (Авторизация)
* `AuthManager.cpp / .h` — Бизнес-логика сессии: проверка паролей (hash), хранение текущего пользователя.
* `Login.cpp / .h` — UI-экран входа.
* `Register.cpp / .h` — UI-экран создания нового сотрудника.
* `UserStruct.h` — Модель данных `User`. Использует `std::string`.

### 📂 `clients/` (Учет абонентов)
* `ClientManager.cpp / .h` — CRUD-операции над базой клиентов.
* `Billing.cpp / .h` — Расчет стоимости услуг, пополнение баланса, история транзакций.
* `ClientStruct.h` — Модель данных `Client` (включая вложенные объекты JSON).

### 📂 `core/` (Системное ядро)
* `Database.cpp / .h` — Менеджер данных. Отвечает за чтение/запись JSON через библиотеку (например, nlohmann/json).
* `InputHandler.cpp / .h` — Обработка нажатий клавиш, валидация ввода, защита от буфера.
* `Render.cpp / .h` — Отрисовка интерфейса (рамки, цвета через WinAPI, таблицы).
* `Utils.cpp / .h` — Хелперы: шифрование паролей, работа с датами, конвертация UTF-8.

### 📂 `data/` (Хранилище JSON)
* `settings.json` — Системные флаги, метаданные.
* `users.json` — База данных сотрудников.
* `clients.json` — База данных клиентов.

## 🛠 Технические стандарты для реализации

### 1. Работа с данными (JSON)
* **Библиотека:** Рекомендуется `nlohmann/json`.
* **Схема:** Все данные считываются в память (std::vector объектов) при старте модуля и сериализуются в файл при сохранении изменений.
* **Безопасность:** Пароли в `users.json` должны храниться только в виде хешей.

### 2. Консольный интерфейс (UI)
* **Кодировка:** Проект должен использовать UTF-8 внутри и корректно выводить кириллицу в консоль (`SetConsoleOutputCP(65001)`).
* **Навигация:** Реализовать через стрелки или цифровое меню с очисткой экрана (`system("cls")` или кастомный рендер).
* **Стилизация:** Использовать псевдографику (символы `╔ ═ ╗`) для отрисовки окон.

### 3. Модели данных (Пример)
Для взаимодействия с JSON модели должны поддерживать макросы сериализации:

```cpp
struct User {
    int id;
    std::string login;
    std::string password_hash;
    int role; // 0 - Admin, 1 - Operator
    bool is_active;

    // Макрос для nlohmann/json
    NLOHMANN_DEFINE_TYPE_INTRUSIVE(User, id, login, password_hash, role, is_active)
};
