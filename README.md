# Automated Working Place (ARM) Console Application

Профессиональное консольное приложение на C++ для автоматизации рабочего места оператора/администратора. Проект разработан с акцентом на качественный пользовательский интерфейс в стиле ретро-терминалов и надежное хранение данных.

## 🚀 Основные возможности

- **Система авторизации**:
  - Регистрация и вход с разделением ролей (Администратор/Оператор).
  - Валидация паролей: минимум 8 символов, наличие цифр, спецсимволов и строго английская раскладка.
  - Первый зарегистрированный пользователь автоматически получает права Администратора.
- **Продвинутый консольный UI**:
  - Интерактивные формы с поддержкой переключения полей через `Tab` и стрелки.
  - Подсветка активных полей ввода (темно-синий фон, каретка `|`).
  - Отрисовка окон с использованием двойных рамок ASCII (`╔═════╗`).
  - Разделители внутри форм для лучшей читаемости.
- **Хранение данных**:
  - Использование формата **JSON** для хранения базы пользователей и клиентов.
  - Интегрирована библиотека `nlohmann/json`.
- **Архитектура**:
  - Разделение на логические модули: `auth` (авторизация), `core` (рендер, ввод, БД), `clients` (управление абонентами).
  - Единый обработчик ввода (`InputHandler`) для абстракции от системных вызовов.

## 🛠 Технологический стек

- **Язык**: C++ (стандарт 11/14/17).
- **Платформа**: Windows (использование WinAPI для управления цветом и курсором консоли).
- **Библиотеки**: `nlohmann/json` (включена в проект как header-only).
- **Сборка**: Visual Studio Solution (.sln).

## 📁 Структура проекта

- `auth/` — логика авторизации и экраны входа/регистрации.
- `clients/` — управление базой абонентов и биллинг.
- `core/` — ядро системы:
  - `Render` — функции отрисовки рамок, текста и инпутов.
  - `InputHandler` — обработка нажатий клавиш (ESC, TAB, ENTER, стрелки).
  - `Database` — загрузка и сохранение данных в JSON.
- `data/` — директория с JSON файлами базы данных.
- `main.cpp` — точка входа и главное меню.

## 🔧 Запуск и сборка

1. Откройте файл решения `.sln` в **Visual Studio 2019/2022**.
2. Убедитесь, что кодировка консоли поддерживает UTF-8 (программа автоматически устанавливает `65001`).
3. Соберите проект (Build Solution).
4. Запустите `kp.exe`.
