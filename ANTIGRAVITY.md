# Project Instructions: C++ ARM (Console Management System)

## Обзор проекта
Ты — экспертный C++ разработчик. Проект представляет собой Автоматизированное Рабочее Место (АРМ) для мобильного оператора — консольное приложение для управления сотрудниками и клиентской базой.
**Ключевая особенность:** Хранение данных в JSON-файлах и работа в среде Windows.

## 🛠 Технический стек
- **Язык:** C++17 (стандарт)
- **Библиотека JSON:** `nlohmann/json` (используется файл `core/json.hpp`)
- **ОС:** Windows (использование WinAPI для консоли, UTF-8 кодировка)
- **Сборка:** Visual Studio (.sln) / MSVC

## Архитектура и структура (Project Map)
Соблюдай строгую модульность, заложенную в проекте:
- `admin/`: `AdminPanel` (меню), `ConfigWizard` (первичная настройка), `LogViewer` (логи).
- `auth/`: `AuthManager` (сессии), `Login`/`Register` (экраны входа), `AuthMenu`.
- `clients/`: `ClientManager` (CRUD), `Billing` (финансы), `ClientStruct.h`.
- `user/`: `HomeMenu` (главное меню оператора), `UserStruct.h` (модель сотрудника).
- `core/`: 
    - `Database.h/cpp`: Центральный менеджер данных.
    - `InputHandler.h/cpp`: Валидация ввода и защита от некорректных данных.
    - `Render.h/cpp`: Отрисовка UI (рамки, таблицы, цвета).
    - `Utils.h/cpp`: Хелперы (хеширование, системные утилиты).
- `data/`: JSON-файлы хранилища (`users.json`, `clients.json`).

## Стандарты кодирования (Rules)
1. **Именование:**
   - Классы: `PascalCase` (напр. `AuthManager`)
   - Методы и переменные: `snake_case` (напр. `check_password`)
   - Константы: `kPascalCase` (напр. `kMaxAttempts`)
2. **Заголовки:** Всегда использовать `#pragma once` во всех `.h`.
3. **Строки:** Использовать только `std::string`. Избегать `char*` и `wchar_t*`, если это не требует WinAPI.
4. **JSON:** Модели должны поддерживать макрос:
   `NLOHMANN_DEFINE_TYPE_INTRUSIVE(Type, field1, field2, ...)`

## Требования к UI и консоли
- **Кодировка:** В `main.cpp` установлена `65001` (UTF-8). Всегда выводи текст с учетом этой кодировки.
- **Отрисовка:** Использовать псевдографику (`╔`, `═`, `╗`) для визуальных блоков. Вызывай методы из `Render.h`.
- **Ввод:** Весь ввод пользователя должен проходить через `InputHandler`, чтобы избежать сбоев потока `cin`.
- **Очистка:** Использовать `system("cls")` для обновления экранов меню.

## Логика данных
1. **Загрузка:** Данные считываются из JSON в `std::vector` соответствующих структур при запуске модуля.
2. **Изменения:** Все манипуляции (добавление, удаление) происходят в векторе в памяти.
3. **Синхронизация:** После любого изменения вектора необходимо вызвать метод сохранения в `Database` для обновления JSON-файла.
4. **Безопасность:** Пароли хранятся исключительно в виде хеш-сумм.

## Инструкции для Агента
- При создании новых методов в `ClientManager` или `AdminPanel`, всегда проверяй права доступа текущего пользователя через `AuthManager`.
- Если функция работает с файлами, всегда проверяй их наличие и валидность JSON-структуры.
- Пиши содержательные комментарии к коду на русском языке.
- При предложении правок старайся минимизировать изменения в `core/`, фокусируясь на бизнес-логике в `admin/` или `clients/`.
